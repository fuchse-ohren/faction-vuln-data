推奨される構成は、不正な HTTP の使用を防ぐことです。
`<LimitExcept>` ディレクティブを利用してメソッドを実行します。

このディレクティブ
ホワイトリスト手法を使用して、HTTP メソッドを許可し、すべてをブロックします。
その他はディレクティブにリストされていないため、すべてのメソッドがブロックされます
改ざんの試み。

最も一般的には、必要な HTTP メソッドのみが、
ほとんどのシナリオは「GET」と「POST」です。これらを許可する例
HTTP メソッドは次のとおりです: `<LimitExcept POST GET> require valid-user</LimitExcept>`
