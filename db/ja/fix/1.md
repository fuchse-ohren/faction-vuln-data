この脆弱性を軽減するには、次のアクションをお勧めします。

1. アプリケーション コードにプリペアド ステートメントまたはパラメータ化されたクエリを実装して、ユーザー入力を SQL ステートメントから分離します。
2. 入力の検証とフィルタリングを実行して、ユーザー入力が期待される形式とデータ型に準拠していることを確認します。
3. プリペアド ステートメントを使用できない場合は、SQL クエリに特殊文字を組み込む前に、ユーザー入力内の特殊文字をエスケープするようにコードベースを確認して更新します。
4. アプリケーションの徹底的なセキュリティ レビューを実施し、既存の SQL インジェクションの脆弱性を特定して修正します。
