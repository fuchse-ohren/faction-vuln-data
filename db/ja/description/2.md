バックドアまたは Web シェルの脆弱性とは、Web アプリケーションまたはシステムのセキュリティ上の欠陥を指し、悪意のあるコードの挿入を通じて影響を受けるシステムへの不正アクセスや制御が可能になります。この脆弱性により、攻撃者は通常の認証プロセスをバイパスしてシステム内に足場を築き、後で悪用できる秘密のエントリ ポイントを効果的に作成することができます。

バックドアはさまざまな方法で導入できますが、一般的な方法の 1 つは、Web アプリケーションのソース コードに悪意のあるコードを挿入することです。このコードは、侵害されたシステム上でコマンドの実行、ファイルのアップロード、またはその他のアクションを実行するように設計されている可能性があります。 Web シェルはバックドアの一種で、通常は Web サーバー上のスクリプトまたはコードの一部として実装され、多くの場合 PHP や ASP などの言語で記述されます。攻撃者が Web シェルにアクセスできるようになると、Web ベースのインターフェイスを通じて侵害されたシステムと対話できるようになり、実質的にシステムのリソースを制御できるようになります。

攻撃者は、次のようないくつかの手段を通じてバックドアまたは Web シェルの脆弱性を悪用する可能性があります。

- __コードインジェクション:__ 攻撃者は、不適切な入力検証や脆弱なセキュリティ制御を利用して、アプリケーションの脆弱な部分に悪意のあるコードを挿入します。

- __ファイルのアップロード:__ アプリケーションが適切な検証なしでファイルのアップロードを許可している場合、攻撃者は悪意のあるファイル (多くの場合 Web シェル スクリプト) をアップロードし、それが実行される可能性があります。

- __構成の問題:__ サーバーまたはアプリケーションの構成が不十分であると、機密ファイルまたはディレクトリが公開され、攻撃者がこれらのリソースを操作できる可能性があります。

- __パッチが適用されていないソフトウェア:__ 既知の脆弱性を持つ古いソフトウェアは、必要なツールと知識にアクセスできる攻撃者によって悪用される可能性があります。

- __サードパーティのプラグイン:__ Web アプリケーションで使用されるサードパーティのプラグインまたは拡張機能の脆弱性を悪用して、不正アクセスが行われる可能性があります。

バックドアまたは Web シェルが配置されると、攻撃者は次のようなさまざまな悪意のあるアクティビティを実行できるようになります。

- __データの盗難:__ ユーザー データ、資格情報、プライベート ドキュメントなどの機密情報にアクセスします。
- __データ操作:__ 侵害されたシステムに保存されているデータを変更、削除、または変更します。
- __分散型サービス拒否 (DDoS):__ 侵害されたシステムを大規模ネットワークの一部として使用して、DDoS 攻撃を開始します。
- __伝播:__ 侵害されたシステムを開始点として使用し、同じネットワーク内の他のシステムを攻撃します。
- __マルウェアの配布:__ 侵害されたアプリケーションにアクセスしている他のユーザーにマルウェアを配布します。
- __アクセスの維持:__ 元の脆弱性にパッチが適用された場合でも、侵害されたシステムに対する継続的な制御を確保します。
