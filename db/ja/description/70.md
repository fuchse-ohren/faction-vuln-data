クロスサイト スクリプティング (XSS) は、Web アプリケーションが他のユーザーに提供される Web ページに信頼できないデータを含めることを許可した場合に発生する、セキュリティ上の脆弱性の一種です。これにより、攻撃者は、他のユーザーが閲覧する Web ページに悪意のあるスクリプト (通常は JavaScript で記述されたもの) を挿入できる可能性があります。悪意のあるスクリプトは被害者のブラウザのコンテキストで実行される可能性があり、さまざまなセキュリティ上の問題や攻撃を引き起こす可能性があります。

保存型 XSS または永続 XSS は、挿入されたスクリプトがターゲット サーバーに永続的に保存され、後でユーザーが特定の Web ページを要求したときに提供されるときに発生します。これは、たとえば、入力が適切にサニタイズおよび検証されていないコメント セクション、フォーラム、またはユーザー プロファイルで発生する可能性があります。他のユーザーが保存されたスクリプトが含まれるページを表示すると、ブラウザーでそのスクリプトが実行され、攻撃者がセッション Cookie や個人データを盗んだり、その他の悪意のあるアクションを実行したりできる可能性があります。
