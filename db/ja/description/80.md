URL のクエリ文字列で機密データを送信すると、いくつかのセキュリティ リスクが生じます。クエリ文字列は、URL の「？」に続く部分です。シンボルであり、通常はキーと値のペアが含まれます。クエリ文字列に機密データを含めることに関連するリスクの一部を次に示します。

1. **可視性**: クエリ文字列はブラウザの履歴やサーバー ログに表示されることが多く、簡単にブックマークできます。パスワードや個人情報などの機密情報がクエリ文字列に含まれている場合、不正アクセスに対してより脆弱になります。
    
2ログと分析. ****：ウェブサーバーおよびさまざまなサードパーティ サービスは、クエリ文字列を含むURL をログに記録する場合があります。これは、機密データがログに保存される可能性があり、それらのログにアクセスできる管理者や攻撃者がそのデータにアクセスできるようになる可能性があることを意味します。
    
3.キャッシュ ****: プロキシ サーバーとキャッシュ メカニズムは、キャッシュ プロセスの一部として、クエリ文字列を含む URL を保存する場合があります。これにより、機密情報がアプリケーションの制御を超えてサーバーに保存されるリスクが生じます。
    
参照ヘッダー4. ****： ユーザーがリンクをクリックすると、クエリ文字列を含む参照ページの URL が HTTP 要求ヘッダーに含まれることがよくあります。クエリ文字列に機密情報が含まれている場合、他の Web サイトに漏洩する可能性があります。
