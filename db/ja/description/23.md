ファイルをウェブサーバーにアップロードする方法にはいくつかの手法があります。その中でも特に有効な方法の一つが HTTP プロトコルの `PUT` メソッドです。このメソッドは主にアプリケーション開発時に使用され、開発者がウェブルートディレクトリ内に直接ファイルをアップロード（いわゆる「置く」操作）することを可能にします。

設計上の特性として、`PUT` メソッドは通常、何らかのフィルタリング機能を備えていません。そのため、サーバー側で実行可能なコード（PHPスクリプトやASPファイルなど）を無審査でアップロードできてしまう可能性があります。

攻撃者は、`PUT` メソッドをサポートしているサーバーを探し出し、これを悪用しようとします。具体的には、既存ページの改ざんや、ウェブシェルと呼ばれる特殊なプログラムをインストールしてサーバーを完全に制御しようとする行為が想定されます。

本ツールによる調査で、この影響を受けたパスではクライアント側から `PUT` メソッドを利用できることが判明しました。このテストでは、実際にウェブルートディレクトリ内にファイルをアップロードし、そのファイルの位置を指定した上で `GET` リクエストを送信することに成功しました。さらに、取得したファイルの内容を検証することで、その真正性を確認しています。
