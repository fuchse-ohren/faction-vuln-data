コード インジェクションの脆弱性は、攻撃者がアプリケーションのコードベースに悪意のあるコードまたはコマンドを挿入し、そのコードがアプリケーション自体によって実行される場合に発生するセキュリティ上の欠陥の一種です。このタイプの脆弱性は通常、不適切な入力検証やアプリケーション内での適切なセキュリティ制御の欠如が原因で発生します。コードインジェクション攻撃は、不正アクセス、データ操作、システム侵害、その他の悪意のあるアクティビティにつながる可能性があります。

__コードインジェクション脆弱性はどのようにして起こるのか:__

- __ユーザー入力:__ 多くのアプリケーションは、テキスト フィールド、クエリ パラメーター、URL パスなどのさまざまな形式でユーザー入力を受け取ります。この入力は、アプリケーションによって動的コンテンツの生成、データベースとの対話、または特定のアクションの実行に使用されます。

- __入力検証の欠如:__ アプリケーションがユーザー入力を適切に検証およびサニタイズしない場合、コード インジェクション攻撃に対して脆弱になります。入力検証には、ユーザーが提供したデータをチェックして、それが予期された形式に準拠しており、悪意のあるコードが含まれていないことを確認することが含まれます。

- __挿入ポイント:__ 攻撃者は、ユーザー入力が適切な検証なしにコードに直接組み込まれるアプリケーション内の場所を特定します。これらのポイントは、「注入ポイント」と呼ばれることがよくあります。一般的なインジェクション ポイントには、SQL クエリ、OS コマンド、JavaScript コードなどが含まれます。

- __悪意のあるペイロード:__ 攻撃者は、アプリケーションによって実行されることを目的としたコードまたはコマンドの断片である悪意のあるペイロードを作成します。これらのペイロードは、注入ポイントに存在する特定の脆弱性を悪用するように設計されています。

- __実行:__ 攻撃者が悪意のあるペイロードの挿入に成功すると、アプリケーションは無意識のうちにそれを正当なコードとして扱い、アプリケーションのコンテキストで実行します。脆弱性とペイロードの性質に応じて、さまざまな種類の攻撃が発生する可能性があります。

__コードインジェクション攻撃の種類:__

- __SQL インジェクション (SQLi):__ 攻撃者は悪意のある SQL クエリを入力フィールドに挿入し、アプリケーションをだまして意図しないデータベース クエリを実行させます。これにより、データベースへの不正アクセスや保存されたデータの操作が行われる可能性があります。
- __コマンド インジェクション:__ 攻撃者は悪意のあるオペレーティング システム コマンドを入力フィールドに挿入し、サーバー上で任意のコマンドを実行できるようにする可能性があります。これにより、システムの侵害や不正アクセスが発生する可能性があります。
- __クロスサイト スクリプティング (XSS):__ 攻撃者は、悪意のあるスクリプトを Web ページに挿入し、ユーザーのブラウザーによって実行されます。これにより、ユーザー セッション データの盗難、マルウェアの拡散、Web ページの改ざんが発生する可能性があります。
- __リモート コード実行 (RCE):__ 攻撃者は、悪意のあるコードを挿入することにより、サーバー上で任意のコードを実行します。これにより、サーバーが完全に制御されたり、データ侵害などが発生したりする可能性があります。
- __サーバー側テンプレートの挿入:__ 攻撃者はテンプレート エンジンの脆弱性を悪用してサーバー側のテンプレートに悪意のあるコードを挿入し、サーバー上でコードを実行し、制御を取得できるようにします。
