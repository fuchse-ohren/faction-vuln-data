TLS 1.0 や TLS 1.1 などの古いバージョンの Transport Layer Security (TLS) プロトコルは、次のような理由から安全ではないと考えられています。

1. 脆弱性: これらの古いバージョンの TLS には、POODLE (パディング Oracle On Downgraded Legacy Encryption) や BEAST (Browser Exploit Against SSL/TLS) などの攻撃に対する脆弱性など、既知のセキュリティ脆弱性があります。これらの脆弱性は、攻撃者によって悪用され、接続を介して送信されるデータを傍受または操作する可能性があります。
2. 弱い暗号スイート: TLS 1.0 および TLS 1.1 は、より弱い暗号化アルゴリズムと暗号スイートをサポートしているため、攻撃者が暗号化を破って機密情報にアクセスすることが容易になります。
3. 最新のセキュリティ機能の欠如: これらの古いバージョンには、暗号化アルゴリズムの改善、完全転送秘密保持、さまざまな種類の攻撃に対する保護の強化など、TLS 1.2 および TLS 1.3 にある最新のセキュリティ機能がありません。
4. コンプライアンスと規制: ペイメント カード業界データ セキュリティ標準 (PCI DSS) や一般データ保護規則 (GDPR) など、多くのセキュリティ標準と規制では、より安全なバージョンの TLS の使用が必要です。
