ブラウザのセキュリティ モデルは通常、Web コンテンツがアクセスできないようにします。
ドメインが別のドメインのデータにアクセスできないようにします。これは一般的に知られています
「同一生成元ポリシー」として。

URL ポリシー ファイルによるクロスドメインの付与
データを読み取るための権限。許可されていない操作は許可されます。
デフォルトで許可されています。 Silverlight の URL ポリシー ファイルは次の場所にあります。
デフォルトでは、ターゲット サーバーのルート ディレクトリに、次の名前が付けられます。
`ClientAccessPolicy.xml` (たとえば、
「www.example.com/ClientAccessPolicy.xml」）。

ドメイン指定の場合
`ClientAccessPolicy.xml` で、サイトは次のことを行う意思があることを宣言します。
そのドメイン内のすべてのサーバーのオペレーターが、
ポリシー ファイルが存在するサーバー上のドキュメント。

の
この Web サイトにデプロイされた `ClientAccessPolicy.xml` ファイルにより、
サーバーからすべてのドメインへ (単一のアスタリスク「*」を純粋なワイルドカードとして使用)
サポートされています）。
