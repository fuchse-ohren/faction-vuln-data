クロスサイト リクエスト フォージェリ (CSRF) は、通常は認証されたユーザーのセッションを利用して、Web アプリケーションがユーザーのブラウザーに持つ信頼を悪用するセキュリティ脆弱性の一種です。 CSRF 攻撃では、攻撃者は、特別に細工されたリンクをクリックするようにユーザーを誘導したり、脆弱なアプリケーションに隠されたリクエストを送信する悪意のあるページにユーザーを誘導したりすることにより、認証されたユーザーをだまして、ユーザーの同意なしに無意識のうちに脆弱な Web アプリケーション上でアクションを実行させます。これらのアクションは、対象となるユーザーの権限に応じて、アカウント設定の変更から金融取引の開始まで多岐にわたります。

CSRF 攻撃の仕組みは次のとおりです。

1. **ユーザー認証:** ユーザーは正規の Web アプリケーションにログインし、セッションを確立します。これには多くの場合、セッション Cookie の取得が含まれます。

2. **攻撃のセットアップ:** 攻撃者は、標的のアプリケーションにリクエストを送信する特別に細工された URL またはフォームを含む悪意のある Web サイトまたは電子メールを作成します。このリクエストには、ユーザー設定の変更、トランザクションの開始、またはユーザーの操作を必要とするその他のアクションの実行が含まれる場合があります。

3. **ユーザーの操作:** 攻撃者は、ユーザーを悪意のある Web サイトにアクセスするか、悪意のあるリンクをクリックするように誘導します。ユーザーがこれを行うと、ブラウザはユーザーのセッション Cookie を含むリクエストを対象のアプリケーションに送信します。

4. **信頼できるコンテキスト:** 対象となるアプリケーションは、正当なリクエストと偽造されたリクエストを区別できないため、有効なセッションを使用してユーザーのブラウザから送信されたリクエストとみなされるため、アクションを実行します。

5. **不正なアクション:** ユーザーのアカウント設定またはその他の機密性の高いアクションが、ユーザーの同意なしに変更されます。これは、アプリケーションがそのアクションがユーザー自身によって開始されたものであると信じ込ませるために発生します。

CSRF 攻撃は、Web アプリケーションがユーザーのブラウザーに持つ信頼を悪用するため、特に危険です。攻撃者はユーザーの資格情報を侵害する必要はありません。代わりに、ユーザーを操作して、知らないうちにユーザーに代わってアクションを実行させます。
