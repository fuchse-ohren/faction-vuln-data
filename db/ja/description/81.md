HTTP リクエストの密輸は、さまざまなコンポーネントが HTTP リクエストを解釈および処理する方法の違いを利用する複雑な攻撃であり、攻撃者がセキュリティ制御をバイパスし、Web アプリケーションの整合性と機密性を侵害する可能性があります。これらの不一致は、多くの場合、リクエスト処理に関与するさまざまなシステム間での解析ルールの違いから発生します。

悪用は多くの場合、「Content-Length」ヘッダーと「Transfer-Encoding」ヘッダーの処理における曖昧さを中心に展開され、メッセージ本文の長さの決定方法に混乱が生じます。

HTTP リクエストの密輸の核心は、フロントエンド サーバーとバックエンド サーバーがこれらのヘッダーを解釈する方法の違いを悪用することにあります。たとえば、攻撃者は Content-Length ヘッダーと Transfer-Encoding ヘッダーが一致しないリクエストを送信し、2 つのサーバー間で混乱を引き起こす可能性があります。フロントエンド サーバーは 1 つのヘッダーを使用してメッセージ長を決定する一方で、バックエンド サーバーはもう 1 つのヘッダーに依存するため、要求の理解に相違が生じる可能性があります。

これらのヘッダーを巧妙に操作することで、攻撃者は解釈の混乱を利用して戦略的にペイロードを挿入することができます。リクエストを慎重に作成することで、攻撃者はセキュリティ対策を超えて追加のリクエストやペイロードをこっそり持ち込んで、不正アクセスを取得したり、アプリケーションが意図していないアクションを実行したりする可能性があります。
