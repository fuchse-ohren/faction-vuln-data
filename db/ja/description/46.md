ブラインド インジェクションは、時間ベースのブラインド SQL インジェクションとも呼ばれ、SQL インジェクション攻撃の一種であり、攻撃者がデータを直接取得したり表示したりすることなく、間接的にデータベースに関する情報を推測できます。この攻撃は、攻撃者が クエリの結果を直接受け取らず、アプリケーションの動作に依存して情報を抽出する必要があるため、「ブラインド」と呼ばれます。ブラインド SQL インジェクションは、アプリケーションが SQL クエリの結果を攻撃者に明らかにしない場合に発生します。

ブラインド SQL インジェクションがどのように機能するかは次のとおりです。

1脆弱なアプリケーション.: 攻撃者は、SQL インジェクションに対して脆弱な Web アプリケーションを特定します。これは通常、適切な検証やサニタイズを行わずに、ユーザーが提供したデータを SQL クエリに含めるアプリケーションです。

2 挿入されたペイロード.： 攻撃者は、悪意のある SQL コードをアプリケーションの入力フィールドまたはパラメーターに挿入します。挿入されたコードは、データベースに関する情報を明らかにするように設計されています。

3目に見えるフィードバックの欠如.: 従来の SQL インジェクションとは異なり、攻撃者は、注入された SQL クエリの成功または失敗を示すアプリケーションからの直接のフィードバックやエラー メッセージを受け取りません。

4時間ベースの推論.: 情報を抽出するために、攻撃者はアプリケーションの応答時間に依存します。攻撃者は、アプリケーションの応答に時間遅延を引き起こす SQL クエリを作成することで、挿入された条件が TRUE か FALSE かを推測できます。

5条件付きクエリ.: 攻撃者は、時間遅延 SQLクエリを使用して、データベースに関する真/偽の質問を提示することで情報を推測する可能性があります。たとえば、攻撃者は次のようなペイロードを使用する可能性があります。

   ```sql
   IF (condition) WAITFOR DELAY '0:0:5' ELSE WAITFOR DELAY '0:0:0'
   ```

条件が true の場合、アプリケーションは応答を 5 秒遅らせます。 false の場合、応答は即時に行われます。

6推論とデータ抽出.: 攻撃者は、さまざまな条件で SQL クエリを繰り返し作成して送信し、情報を少しずつ抽出します。この情報には、データベース スキーマの詳細、テーブル名、さらにはデータ値が含まれる場合もあります。

ブラインド SQL インジェクションは、攻撃者が情報を抽出するために間接的な方法に依存する必要があるため、標準の SQL インジェクションと比較してより複雑で時間がかかる攻撃です。データベースの構造と内容を推測するには、忍耐と注意深くクエリを作成する必要があります。ブラインド SQL インジェクションを防ぐには、Web 開発者は安全なコーディング慣行に従い、プリペアド ステートメントを使用し、入力検証を実行する必要があります。さらに、エラー メッセージと応答時間を制限すると、攻撃者によるこの脆弱性の悪用がより困難になる可能性があります。
