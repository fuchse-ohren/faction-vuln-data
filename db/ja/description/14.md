Concurrent Version System (CVS) と Subversion (SVN) は、開発者が時間の経過とともに進化するコードのさまざまなバージョンを管理するために使用するツールです。

場合によっては、バージョン管理システムが開発者またはユーザーの情報をコード内に正しく保存しないことがあります。このような場合、この情報は、HTML 内で直接、またはコード コメント内で、意図せずエンド ユーザーに表示される可能性があります。情報収集の前段階として、悪意のある攻撃者が Web サイトをクロールし、自動技術を使用してページ上に存在する CVS/SVN 関連情報を特定する可能性があります。

この情報は、デプロイされたアプリケーションに関する洞察を提供するため、攻撃者にとって貴重になる可能性があります。バージョンの詳細が明らかになり、攻撃者がアプリケーションのテクノロジー スタックを理解するのに役立つ可能性があります。さらに、このデータはさらなる偵察やソーシャル エンジニアリング攻撃に寄与する可能性があります。

同様の自動化された方法を使用したスキャン プロセスでは、影響を受けるページに CVS または SVN の詳細が保存されているインスタンスを特定できました。
