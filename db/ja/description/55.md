クロスサイト スクリプティング (XSS) は、Web アプリケーションが他のユーザーに提供される Web ページに信頼できないデータを含めることを許可した場合に発生する、セキュリティ上の脆弱性の一種です。これにより、攻撃者は、他のユーザーが閲覧する Web ページに悪意のあるスクリプト (通常は JavaScript で記述されたもの) を挿入できる可能性があります。悪意のあるスクリプトは被害者のブラウザのコンテキストで実行される可能性があり、さまざまなセキュリティ上の問題や攻撃を引き起こす可能性があります。

リフレクト XSS 攻撃では、挿入されたスクリプトはターゲット サーバーに永続的に保存されません。代わりに、Web サーバーまたはアプリケーションに反映され、ユーザーのブラウザーのコンテキストですぐに実行されます。これは通常、ユーザーが悪意のあるリンクをクリックしたり、悪意のあるスクリプトを含む特別に作成された URL を操作したりしたときに発生します。その後、スクリプトは現在のユーザーのセッションのコンテキストで実行されます。
